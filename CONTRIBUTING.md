# Contributing to go-auth

## Getting Started

1. Fork the repo and clone your fork
2. Run `go mod download` to install dependencies
3. Run `go test ./...` to confirm everything passes

## Making Changes

- Create a branch: `git checkout -b feat/your-feature`
- Add tests for any new functionality
- Run `go test ./...` and `go fmt ./...` before submitting

## Commit Messages

We use [Conventional Commits](https://www.conventionalcommits.org/):

- `feat:` new feature
- `fix:` bug fix
- `docs:` documentation
- `test:` tests only
- `refactor:` no behaviour change

## Pull Requests

- Reference the issue your PR closes (`Closes #N`)
- Describe what the PR does and how you tested it
- Keep PRs focused — one thing per PR

## Reporting Bugs

Open an issue and include your Go version, OS, and a minimal reproduction case.
